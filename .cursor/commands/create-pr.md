# Create Pull Request

Create a well-documented pull request for changes to kheMessage.

## Pre-PR Checklist

1. **Code complete** - All changes implemented and tested
2. **Self-review done** - Reviewed your own changes
3. **Tests pass** - Manual testing completed
4. **Documentation updated** - README updated if needed

## PR Title Format

Use conventional commit style:
- `feat: add new feature description`
- `fix: resolve bug description`
- `docs: update documentation`
- `style: formatting changes`
- `refactor: code restructuring`
- `perf: performance improvement`

## PR Description Template

```markdown
## Summary

Brief description of what this PR does.

## Changes

- Change 1
- Change 2
- Change 3

## Testing

- [ ] Tested in Chrome
- [ ] Tested in Firefox
- [ ] Tested in Safari
- [ ] Tested on mobile
- [ ] Tested offline (PWA)
- [ ] Tested both themes

## Screenshots

If UI changes, add before/after screenshots.

## Notes

Any additional context or considerations.
```

## Branch Naming

- `feat/feature-name` - New features
- `fix/bug-description` - Bug fixes
- `docs/what-changed` - Documentation
- `refactor/what-changed` - Refactoring

## Review Process

1. Create PR with description
2. Request review if collaborating
3. Address feedback
4. Squash and merge when approved
